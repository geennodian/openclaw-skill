# Node.js Todo CLI アプリ

## 概要

コマンドラインから操作できるシンプルなTodoアプリ。タスクの追加・一覧表示・完了マーク・削除をサポートし、データをローカルのJSONファイルに永続化する。外部依存を最小限に抑え、Node.js標準モジュールのみで実装する。

## 要件

- タスクを追加できる（テキスト指定）
- タスクの一覧を表示できる（ID・タイトル・完了状態・作成日時）
- タスクをIDで完了状態にマークできる
- タスクをIDで削除できる
- データは `data/todos.json` に保存され、プロセス終了後も維持される
- 無効なコマンドや存在しないIDに対してわかりやすいエラーメッセージを表示する

## 技術方針

- **ランタイム**: Node.js (v18以上)
- **外部ライブラリ**: なし（`fs`, `path` など標準モジュールのみ）
- **データ形式**: JSON（配列でTodoオブジェクトを管理）
- **エントリポイント**: `todo.js`（`node todo.js <command> [args]` で実行）
- **アーキテクチャ**: 単一ファイル構成。コマンドパース → ストレージ読み書き → 処理 → 出力 の直列フロー

## ファイル構成

```
todo.js          # メインスクリプト（エントリポイント兼全ロジック）
data/
  todos.json     # 永続化データ（初回実行時に自動生成）
```

## インターフェース定義

### CLIコマンド

```
node todo.js add <title>       # タスクを追加
node todo.js list              # タスク一覧を表示
node todo.js done <id>         # タスクを完了にする
node todo.js delete <id>       # タスクを削除する
```

### Todoオブジェクト（JSON構造）

```json
{
  "id": 1,
  "title": "タスクのタイトル",
  "done": false,
  "createdAt": "2026-03-30T00:00:00.000Z"
}
```

### 内部関数

```js
loadTodos(): Todo[]
// data/todos.json を読み込んで配列を返す。ファイルが存在しない場合は []

saveTodos(todos: Todo[]): void
// todos 配列を data/todos.json に書き込む

addTodo(title: string): void
doneTodo(id: number): void
deleteTodo(id: number): void
listTodos(): void
```

### 出力フォーマット（list）

```
[1] [ ] タスクA  (2026-03-30)
[2] [x] タスクB  (2026-03-30)
```

## 実装上の注意点

- `data/` ディレクトリが存在しない場合は `fs.mkdirSync` で自動生成する
- IDは配列内の最大ID + 1 で採番する（削除後の欠番を再利用しない）
- 存在しないIDを指定した場合、`Error: ID <n> not found` を表示して終了コード1で終了する
- タイトルが空または未指定の場合、`Error: title is required` を表示して終了コード1で終了する
- `JSON.stringify(todos, null, 2)` でインデント付き書き込みにする
- `list` でタスクが0件の場合は `No todos.` を表示する

## 完了条件

- `add` でタスクが追加され `data/todos.json` にデータが書き込まれる
- `list` で全タスクがID・完了状態・タイトル・日付付きで表示される
- `done <id>` で対象タスクが完了状態になる
- `delete <id>` で対象タスクがリストから消える
- 存在しないIDでエラーメッセージが表示され終了コードが1になる
- `data/todos.json` が存在しない状態でも初回実行が正常に動作する
